---

### 1.2. 技术选型原因说明

#### 🔹 后端：Python + FastAPI

项目的后端采用 Python 搭配 FastAPI，主要基于以下考虑：

- Todo-List 属于 **轻量级项目**，不需要复杂的后端架构。
- Python **语法简洁、开发效率高**，适合快速迭代。
- FastAPI 是 **轻量级、高性能、现代化**的 Python Web 框架，支持自动生成 API 文档，并对异步处理和 RESTful 设计有良好支持。
- 在本项目中，FastAPI 负责定义 **增删改查（CRUD）接口**，用于处理前后端的数据交互，使开发过程清晰高效。

该技术组合使得项目能在短时间内完成，并具备良好的结构与扩展性。

---

#### 🔹 前端：Vue 3

选择 Vue 3 作为前端框架的原因包括：

- 相比 React，Vue 的语法更简洁，开发体验更容易上手。
- Vue 生态统一，不需要额外安装过多额外库即可完成开发。
- 适合 Todo-List 这样结构明确、逻辑相对简单的轻量级应用。

因此，Vue 3 更适合本项目的功能需求与开发节奏。

---

#### 🔹 数据库：SQLite

选择 SQLite 作为存储方案的原因：

- Todo-List 项目的数据量较小，SQLite 足以满足需求。
- SQLite **无需复杂环境配置**，非常适合轻量级项目。
- 数据库以单个 `.db` 文件形式存在，便于在本地或部署时使用。
- 面试官下载 GitHub 项目后，无需安装 MySQL 或 PostgreSQL，即可立即运行项目。

---

## 2. 项目结构设计

### 2.1 整体架构说明
本项目采用前后端分离架构：

- **前端（Vue3 + TypeScript）**：负责页面展示、交互逻辑、动画效果以及调用后端 API。
- **后端（FastAPI + Python）**：处理业务逻辑、提供 RESTful API、数据库增删改查。
- **数据库（SQLite）**：用于数据持久化存储，轻量级、无需额外配置，适合个人项目与快速开发。

前端通过 Axios 请求后端提供的接口，并与 SQLite 数据进行交互，整体设计结构清晰，逻辑分离明确。

## 3. 需求细节与决策

### 3.1. 字段输入要求
- 添加待办任务时：
  - **标题（必填）**：如果用户未填写标题，则不会创建任务。
  - **内容与截止日期（选填）**
  - **任务类别与优先级（可选）**：若未选择，将采用系统默认设置。

---

### 3.2. 已完成任务展示方式
- 在任务列表中，已完成的任务会使用 **降低透明度或模糊显示**方式标记。
- 完成的任务会同步进入 **已完成任务列表**。

---

### 3.3. 任务排序逻辑
由于项目支持按类别分组显示，因此排序逻辑按以下规则进行：

1. **列表排序优先级规则：**
   1. 生活类任务（Life）
   2. 学习类任务（Study）
   3. 工作类任务（Work）

2. **每个类别内部排序方式：**
   - 优先按任务优先级（P1 → P2 → P3 → P4）
   - 若优先级相同，则按任务创建时间排序

>  任务是否超时或剩余时间不会影响排序，只用于 UI 提示效果。

---

### 3.4. 扩展功能设计

项目除了实现基础功能外，还支持额外的扩展能力：

####  功能 1：任务搜索
- 用户可以在某个任务列表中输入关键词。
- 系统将实时筛选并展示标题中包含关键词的任务。

####  功能 2：元气值系统（特色交互功能）
- 用户在**任务截止日期前**完成任务可获得 **+5 元气值**。
- 用户在**任务截止日期前**没有完成任务会得到惩罚 **-5 元气值**。
- 元气值会触发动画与界面反馈，包括：
  - 表情变化
  - 激励语更新
  - 元气值提升特效

> ✨ 当元气值达到 **10 的倍数**时，会额外触发带有 “元气值 UP” 的特殊鼓励动画。

经过对比，我最终选择：

### 🧩 代码片段生成与辅助

#### 后端部分  
我让 AI 帮我：

- 搭建 FastAPI 基础架构
- 生成 RESTful API 端点

在此基础上，我自己实现了业务逻辑层，使我可以专注使用 Python 处理数据和逻辑，提高效率。

---

### 🛠️ AI 生成内容的修改方式

AI 生成的前端代码虽然可运行，但存在一些问题，例如：

| 问题类型 | 示例 |
|---------|------|
| 布局不符合交互逻辑 | 输入框与按钮排列顺序不合理 |
| UI 不够美观 | 元素大小比例不协调 |
| 可读性问题 | 某些文字颜色与背景色相同 |
| 逻辑缺陷 | 元气值动态表情和激励语刷新逻辑错误 |

针对这些问题，我通过查看前端代码，对元素的位置、大小、颜色等属性进行了微调，使页面效果符合预期。
此外，AI 生成的元气值表情变化逻辑也存在问题。
我通过修改前端代码，使得元气值显示条下方的表情和激励语的变化符合预期。

### 📌 本地运行方式

#### 🔧 环境依赖

| 模块 | 技术栈 |
|------|--------|
| 后端 | FastAPI、SQLModel、Uvicorn |
| 前端 | Node、npm、Vue、TypeScript、Axios、Vite、GSAP |

### ▶️ 启动步骤

#### **1）后端运行方式**

在 PyCharm 中配置 `uvicorn` 运行参数：

1. 顶部菜单栏选择 **Edit Configurations**
2. 点击左上角 **+**，选择 **Python**
3. 设置：

| 配置项 | 内容 |
|--------|------|
| Module name | `uvicorn` |
| Parameters | `main:app --reload` |
| Working directory | `./backend` |

4. 点击 **Run** 按钮启动后端服务。

---

#### **2）前端运行方式**

打开终端并执行：

```bash
cd frontend
npm install
npm run dev
```
